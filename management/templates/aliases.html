Алиасы

Добавить почтовый алиас

Алиасы — это почтовые перенаправления. Алиас может пересылать почту почтовому пользователю или на любой email-адрес.

Чтобы отправлять исходящую почту от имени алиаса или другого адреса, отправляющий пользователь должен быть добавлен в список разрешённых отправителей для этого алиаса.

Catch-All алиас принимает всю почту домена, которая не совпала с другими адресами. Алиас домена пересылает всю иначе не совпавшую почту с одного домена на другой, сохраняя часть адреса до знака @.

API почтовых алиасов (для опытных пользователей)

Используйте API почтовых алиасов, чтобы добавлять и удалять алиасы из командной строки или собственных сервисов.

Использование:

curl -X VERB [-d "parameters"] --user {email}:{password} https://{{hostname}}/admin/mail/aliases[action]

Квадратные скобки обозначают необязательный аргумент. Обратите внимание: тело POST-запроса parameters должно быть URL-кодировано.

Email и пароль, переданные в параметре --user, должны принадлежать административному пользователю этой системы.

Команды

Метод Действие
GET(нет) Возвращает список существующих почтовых алиасов. Добавление ?format=json к URL вернёт результат в JSON.
POST/add Добавляет новый почтовый алиас. Обязательные параметры POST-запроса: address и forwards_to.
POST/remove Удаляет почтовый алиас. Обязательный параметр POST-запроса: address.

Примеры:

Попробуйте эти примеры. Для простоты в них не указан аргумент --user me@mydomain.com:yourpassword; его нужно заполнить вашим email-адресом и паролем.

# Gives a JSON-encoded list of all mail aliases
curl -X GET https://{{hostname}}/admin/mail/aliases?format=json

# Adds a new alias
curl -X POST -d "address=new_alias@mydomail.com" -d "forwards_to=my_email@mydomain.com" https://{{hostname}}/admin/mail/aliases/add

# Removes an alias
curl -X POST -d "address=new_alias@mydomail.com" https://{{hostname}}/admin/mail/aliases/remove
