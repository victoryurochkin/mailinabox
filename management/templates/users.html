Пользователи

Добавить почтового пользователя

Добавьте email-адрес в систему. Для него будет создана отдельная учётная запись и пароль.

  • Парольs must be at least eight characters consisting of English letters and numbers only. For best results, generate a random password.
  • Use aliases to create email addresses that forward to existing accounts.
  • Администраторs get access to this control panel.
  • User accounts cannot contain any international (non-ASCII) characters, but aliases can.
  • Квотаs may not contain any spaces, commas or decimal points. Suffixes of G (gigabytes) and M (megabytes) are allowed. Для неограниченного объёма укажите 0 (ноль).

Существующие почтовые пользователи

Email-адрес Size Used Квота Actions
set quota | set password | archive account
To restore account, create a new account with this email address. Or to permanently delete привилегию mailbox, delete привилегию directory on привилегию machine.

API почтовых пользователей (для опытных пользователей)

Используйте API почтовых пользователей, чтобы добавлять, изменять и удалять пользователей из командной строки или собственных сервисов.

Использование:

curl -X VERB [-d "parameters"] --user {email}:{password} https://{{hostname}}/admin/mail/users[action]

Квадратные скобки обозначают необязательный аргумент. Обратите внимание: тело POST-запроса parameters должно быть URL-кодировано.

Email и пароль, переданные в параметре --user, должны принадлежать административному пользователю этой системы.

Команды

Verb Action
GET(none) Returns a list of existing mail users. Adding ?format=json to привилегию URL will give JSON-encoded results.
POST /add Adds a new mail user. Required POST-body parameters are email and password. Optional parameters: privilege=admin and quota
POST /remove Removes a mail user. Required POST-by parameter is email.
POST/privileges/add Used to make a mail user an admin. Required POST-body parameters are email and privilege=admin.
POST/privileges/remove Used to remove привилегию admin privilege from a mail user. Required POST-body parameter is email.
GET /quota Get привилегию quota for a mail user. Required POST-body parameters are email and will return JSON result
POST /quota Set привилегию quota for a mail user. Required POST-body parameters are email and quota.

Примеры:

Попробуйте эти примеры. Для простоты в них не указан аргумент --user me@mydomain.com:yourpassword; его нужно заполнить административным email-адресом и паролем.

# Gives a JSON-encoded list of all mail users
curl -X GET https://{{hostname}}/admin/mail/users?format=json

# Adds a new email user
curl -X POST -d "email=new_user@mydomail.com" -d "password=s3curE_pa5Sw0rD" https://{{hostname}}/admin/mail/users/add

# Removes a email user
curl -X POST -d "email=new_user@mydomail.com" https://{{hostname}}/admin/mail/users/remove

# Adds admin privilege to an email user
curl -X POST -d "email=new_user@mydomail.com" -d "privilege=admin" https://{{hostname}}/admin/mail/users/privileges/add

# Removes admin privilege from an email user
curl -X POST -d "email=new_user@mydomail.com" https://{{hostname}}/admin/mail/users/privileges/remove
