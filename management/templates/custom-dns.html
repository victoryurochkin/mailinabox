Использование вторичного DNS-сервера

Если доменная зона требует два отдельных DNS-сервера, можно либо настроить внешний DNS и полностью игнорировать DNS-сервер этого Mail-in-a-Box, либо использовать DNS-сервер на этом сервере и добавить вторичный DNS-сервер.

Если вы используете вторичный DNS-сервер, нужно выбрать провайдера такой услуги. Её может предоставлять регистратор домена или облачный провайдер. После настройки укажите ниже hostname вторичного DNS-сервера провайдера, а не его IP-адрес.

Несколько вторичных серверов можно разделять запятыми или пробелами, например ns2.hostingcompany.com ns3.hostingcompany.com. Чтобы разрешить передачу зоны дополнительным серверам без указания их как вторичных DNS-серверов, добавьте перед hostname, IP-адресом или подсетью префикс xfr:, например xfr:10.20.30.40 или xfr:10.0.0.0/8.

Очистите поле выше и нажмите «Обновить», чтобы использовать сам этот сервер как вторичный DNS. Это стандартная настройка по умолчанию.

API пользовательских DNS-записей

Используйте DNS API этого сервера для настройки пользовательских DNS-записей доменов, размещённых здесь. Например, так можно создать собственный сервис динамического DNS.

Использование:

curl -X VERB [-d "value"] --user {email}:{password} https://{{hostname}}/admin/dns/custom[/qname[/rtype]]

Квадратные скобки обозначают необязательный аргумент.

Команды

Метод
Использование
GET Возвращает подходящие пользовательские DNS-записи в виде JSON-массива объектов. Каждый объект содержит ключи qname, rtype и value. Необязательные параметры qname и rtype в URL запроса фильтруют возвращаемые записи. Тело запроса (-d "...") должно отсутствовать.
PUT Задаёт пользовательскую DNS-запись, заменяя существующие записи с теми же qname и rtype. Используйте PUT вместо POST, когда для пары qname и rtype должно быть только одно значение, например для обычных записей A без round-robin.
POST Добавляет новую пользовательскую DNS-запись. Используйте POST, когда нужно несколько записей TXT или round-robin записи A. PUT в таком случае удалил бы ранее добавленные записи.
DELETE Удаляет пользовательские DNS-записи. Если тело запроса (-d "...") пустое или отсутствует, будут удалены все записи, совпадающие по qname и rtype. Если тело запроса передано, будет удалена только запись, совпадающая по qname, rtype и значению.

Параметры

Параметр Значение
email Email-адрес любого административного пользователя этой системы.
password Пароль этого пользователя.
qname Полное доменное имя записи, которую вы настраиваете. Оно должно быть одним из доменных имён или поддоменом одного из доменов, размещённых на этом сервере. Чтобы добавить новые домены, создайте почтовых пользователей или алиасы.
rtype Тип ресурсной записи. Если параметр не указан, используется A. Возможные значения: A (IPv4-адрес), AAAA (IPv6-адрес), TXT (текстовая строка), CNAME (алиас в виде полного доменного имени; не забудьте точку в конце), MX, SRV, SSHFP, CAA или NS.
value Для PUT, POST и DELETE — значение записи. Если rtype равен A или AAAA, а value пустой или отсутствует, будет использован IPv4- или IPv6-адрес удалённого хоста. Для curl используйте параметры -4 или -6. Это удобно для динамического DNS.

Строгие записи SPF и DMARC будут добавлены ко всем пользовательским доменам, если вы не переопределите их вручную.

Примеры:

Попробуйте эти примеры. Для простоты в них не указан аргумент --user me@mydomain.com:yourpassword; его нужно заполнить вашим email-адресом и паролем.

# sets laptop.mydomain.com to point to the IP address of the machine you are executing curl on
curl -X PUT https://{{hostname}}/admin/dns/custom/laptop.mydomain.com

# deletes that record and all A records for that domain name
curl -X DELETE https://{{hostname}}/admin/dns/custom/laptop.mydomain.com

# sets a CNAME alias
curl -X PUT -d "bar.mydomain.com." https://{{hostname}}/admin/dns/custom/foo.mydomain.com/cname

# deletes that CNAME and all CNAME records for that domain name
curl -X DELETE https://{{hostname}}/admin/dns/custom/foo.mydomain.com/cname

# adds a TXT record using POST to preserve any previous TXT records
curl -X POST -d "some text here" https://{{hostname}}/admin/dns/custom/foo.mydomain.com/txt

# deletes that one TXT record while preserving other TXT records
curl -X DELETE -d "some text here" https://{{hostname}}/admin/dns/custom/foo.mydomain.com/txt
